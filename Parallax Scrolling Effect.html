The Technique

Layers moving at different speeds create the illusion of depth

Parallax scrolling works by translating each layer along the Y axis at a fraction of the scroll offset. Stars move slowly — only 10% of the scroll distance. Near trees move at 75%. The human eye interprets this differential motion as real three-dimensional space.

How It Works

Scroll events drive CSS transforms in real time

On every scroll event, we read window.scrollY and apply translateY(offset × speed) to each layer. Using will-change: transform promotes layers to the GPU, keeping animation buttery at 60 fps.

Implementation

The math is beautifully simple

Each layer has a data-speed attribute between 0 and 1. On scroll: layer.style.transform = translateY(scrollY × speed × -1). Negative direction moves layers upward relative to their parent, creating the parallax offset. An IntersectionObserver handles the fade-in animations independently.

Performance considerations

We use requestAnimationFrame to throttle updates to the display refresh rate, and will-change: transform to hint the browser to create separate compositor layers — preventing unnecessary repaints and keeping the main thread free.
